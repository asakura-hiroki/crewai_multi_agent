# 要件定義書：ブロック崩しゲーム（機能拡張およびバグ修正）

## 1. プロジェクト概要
本プロジェクトは、既存のPython実装による「ブロック崩しゲーム」の動作安定化（SyntaxErrorの解消）および、ゲーム性向上のためのメカニクス（再開機能、難易度上昇ロジック）の実装を目的とする。

## 2. 修正・実装対象範囲
### 2.1 バグ修正（Critical）
プログラムの実行時における構文エラー（SyntaxError）を解消し、デルタタイム（dt）に基づいたフレームレートに依存しない滑らかな移動計算を復元する。

### 2.2 新機能実装（Functional）
1.  **リスタート・レジューム機能**: `R`キー入力によるゲーム状態の再開/リセット。
2.  **難易度動的変化ロジック**: スコア加算に応じたボール移動速度の自動加速。

---

## 3. 機能要件

### 3.1 実行時エラーの解消（Syntax Error Fix）
**【対象箇所】**
`program.py` line 297付近

**【修正内容】**
不正な一行に結合された代入文を、正しい変数定義と時間差計算のロキックに分離する。
*   **誤（修正前）:** `dt = current import_dt = current_time - self.last_time`
*   **正（修正後）:** 
    ```python
    current_time = time.time() # または適切な時間取得メソッド
    dt = current_time - self.last_time
    self.last_time = current_time
    ```
*   **目的:** `dt` (delta time) を正しく算出することで、PCのスペックに関わらずボールやパドルの移動速度を一定（フレームレート非依存）にする。

### 3.2 ゲーム・メカニクス（Game Mechanics）
#### A. スコア連動型加速システム
*   **トリガー:** スコアが100点の倍数（100, 200, 300...）に達した瞬間。
*   **挙動:** ボールの移動ベクトル（速度ベクトル）の大きさを増幅させる。
*   **計算式例:** `current_velocity = current_velocity * 1.1` （※倍率は実装上、調整可能な定数として定義すること）
*   **注意点:** スコアが更新された「瞬間」に判定を行うこと。毎フレーム判定を行うと、同一スコア内で指数関数的に加速してしまうため、`previous_score` との比較、または剰余演算を用いたフラグ管理を行うこと。

#### B. キー入力による状態制御（Rキー）
*   **トリガー:** `R` キーの押下。
*   **挙動:**
    *   **Game Over/Pause 状態時:** ゲームの初期状態（パドルの初期位置、ボールの初期位置、スコア0、ブロックの全配置）へリセットし、ゲームを再開する。
    *   **実行中:** 状態を初期化し、リスタートさせる。

## 4. 非機能要件

### 4.1 動作安定性
*   フレームレート（FPS）が変動しても、ボールの移動距離が `dt` に比例して一定に保たれること。

### 4.2 コードの保守性
*   加速倍率（例：1.1倍）や、加速トリガーとなるスコア閾値（100点）は、マジックナンバーを避け、クラス変数または定数として定義すること。
