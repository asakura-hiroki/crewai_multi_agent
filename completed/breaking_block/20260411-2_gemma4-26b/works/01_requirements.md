# 要件定義書：ブロック崩しゲーム（改修版）

## 1. プロジェクト概要
本プロジェクトは、Python（Pygame想定）を用いたブロック崩しゲームの開発である。過去のプロトタイプにおいて発生した、構文エラー（SyntaxError）、タイポ、および表示・操作上の不具合をすべて解消し、指示された仕様通りの視覚効果とゲーム性を備えた完成版を構築する。

## 2. 基本仕様（Core Mechanics）
### 2.1 ゲーム要素
* **ボール (Ball):** 
    * 形状：**白色の円（●）**。
    * 特徴：スコアアップに伴い、徐々に移動速度が加速する。
* **バー (Paddle):** 
    * 形状：画面下部に水平に配置される矩形。
    * 操作：キーボードの**左右矢印キー（Left/Right）**により左右に移動。
    * 応答性：操作の遅延（ラグ）がない、クイックな動きを実現すること。
* **ブロック (Blocks):** 
    成立条件：**5段の階層構造**を持つ。
    視覚効果：**各層（レイヤー）ごとに異なる色**で塗り分ける（カラフルな状態）。
    消失条件：ボールが接触すると消滅する。

### 2.2 ゲームルール
* **レベルアップ:** 
    * 一つの層（全ブロック）をすべて消去すると、レベルが上昇する。
    * レベル上昇時に「Level Up」等の表示、またはレベル表示の更新を行う。
* **加速メカニズム:** 
    * スコアが**100点加算されるごとに、ボールの移動速度を上昇**させる。
* **ゲームオーバー:** 
    * ボールが画面下端（バーの範囲外）に到達した際、ゲームオーバー状態とする。
    * **重要：** ゲームオーバー時においても、**「ブロック」「バー」「Score」「Level」の表示が消えずに画面上に残り続けること**。
* **リトライ機能:** 
    * **Rキー**を押下することで、現在の状態をリセットし、ゲームを最初から再開（Retry）できること。

## 3. UI/UX 仕様
* **スコア表示 (Score):** 画面上にリアルタイムで表示。
* **レベル表示 (Level):** 
    * 画面上に表示。
    * 配置：**画面の右端（Right-aligned）**に配置すること。
* **視覚的一貫性:** 
    * 描画エラー（表示の消失、色の塗り分け不足）を完全に排除し、指示通りの色使いを維持する。

## 4. 技術的制約・非機能要件（重要：エラー防止）
エンジニアは実装にあたり、以下の過去の不具合（SyntaxError等）を徹底的に排除すること。

### 4.1 構文エラーの排除リスト
* **タイポの禁止:** 
    * 全角文字（`．` など）がコード内に混入していないこと。
    * 変数名の打ち間違い（例：`3alen_WIDTH` のような、数字から始まる不適切な変数名やタイポ）を排除すること。
* **構文規則の遵守:**
    * **代入式（Walrus operator `:=`）の誤用禁止:** `self.padding_limit := ...` のように、クラス属性に対して代入式を使用し、SyntaxErrorを引き起こす実装を避けること。適切な変数割り当てを行うこと。
    * **if文の構文チェック:** `if` 文の条件式において、括弧の閉じ忘れや、カンマの欠落、不完全な論理演算（`and` の後の条件不足）がないこと。
    * **リテラルエラーの排除:** 数値リテラルの不適切な記述（`invalid decimal literal`）を避けること。
* **ロジックの正確性:**
    * 衝突判定（Collision Detection）のロジックにおいて、`ball.y + ball.radius` 等の境界判定が正しく、構文的に成立していることを確認すること。

### 4.2 パフォーマンス
* **入力レスポンス:** バーの移動速度が遅いという指摘があるため、イベントループ内での移動処理がスムーズに行われるよう、更新頻度と移動量を適切に調整すること。

## 5. 完了定義（Definition of Done）
1. [ ] プログラムがエラーなく起動し、実行可能である。
2. [ ] 5段のカラフルなブロックが正しく表示されている。
3. [ ] ボールは白い円であり、スコアに応じて加速する。
4. [ ] バーが左右キーで遅延なく操作できる。
5. [ ] 100点ごとに加速し、層ごとにレベルが上がる。
6. [ ] Rキーでゲームがリトライできる。
7. [ ] ゲームオーバー時も、スコアやブロック等の要素が画面から消えない。
8. [ ] Level表示が画面右側に正しく配置されている。
