# テトリスゲーム・プログラム修正・機能強化：詳細仕様書・設計書

## 1. プログラム概要
本ドキュメントは、Pythonおよび`tkinter`を用いたテトリスゲームの、バグ修正および機能拡張に関する詳細な仕様と設計を定義するものである。

## 2. 機能仕様書 (Functional Specifications)

### 2.1 ゲームループ・ロジック
| 機能名 | 詳細仕様 | 補足・条件 |
| :--- | :--- | :--- |
| **ピース生成** | ゲーム開始時およびライン消去後、新しいピースを生成する。 | 既存のブロックと衝突する場合は即座にGame Over。 |
| **落下処理** | 一定間隔（Tick）ごとにピースを1セル下方へ移動させる。 | 壁または設置済みブロックに衝突した場合、設置を確定。 |
| **ライン消去** | グリッドの横一行がすべてブロックで埋まった際、その行を削除し、上の行をすべて一段下げる。 | 消去時にスコアを加算。 |
| **衝突判定** | ピースの移動・回転時に、グリッドの境界外または既存ブロックとの重なりを判定する。 | 判定ロジックは `check_collision` メソッドに集約。 |
| **Game Over 判定** | 1. 新しいピース生成時に、その初期位置が既にブロックで埋まっている。<br>2. ピースが最上段に到達し、移動・回転が不可能な状態。 | 判定後、`game_over` フラグを `True` にし、UIを更新。 |
| **リセット機能** | `R` キー押下時、全ての変数（Score, Level, Grid, Pieces）を初期状態に回帰させる。 | 既存のキャンバス表示もクリアすること。 |

### 2.2 ユーザーインターフェース仕様 (UI Specifications)
#### 2.2.1 レイアウト構成
1.  **Top Area (Score/Level Display)**
    *   ブロックフィールドの外（上部）に配置。
    *   `Label` ウィジェットを使用。背景色は `bg="#222"` 等の統一された色。
2.  **Main Area (Play Field)**
    *   `tkinter.Canvas` を使用。
    *   ブロックの描画、および Game Over 時のテキストオーバーレイ。
3.  **Side Area (Info/Status)**
    *   ゲームの進行状況（現在のレベル等）を表示。
    *   背景色の指定に重複（`bg` の重複引数）を禁止する。

#### 2.2.2 Game Over 表示仕様
*   **表示位置**: キャンバスの座標 `(width/2, height/2)` を中心にテキストを配置。
*   **表示内容**:
    *   Line 1: "Game Over" (太字、大きく表示)
    *   Line 2: "Press R to Restart" (直下に配置)
*   **描画手法**: Canvasの `create_text` メソッドを使用。

### 2.3 入力仕様
| キー | アクション |
| :--- | :--- |
| `Left Arrow` | ピースを左へ移動 |
| `Right Arrow` | ピースを右へ移動 |
| `Down Arrow` | ピースを加速落下 |
| `Up Arrow` | ピースを回転 |
| `R` | ゲームの全状態をリセットして再開 |

## 3. 基本設計書 (Basic Design)

### 3.1 クラス構造設計

#### Class: `TetrisGame`
ゲームのメインエンジンとなるクラス。

**Attributes (メンバ変数):**
*   `self.root`: `tkinter.Tk` インスタンス
*   `self.canvas`: `tkinter.Canvas` インスタンス
*   `self.grid`: `List[List[str]]` (20x10のグリッド。空は `None` または `""`)
*   `self.score`: `int`
*   `self.level`: `int`
*   `self.game_over`: `bool`
*   `self.current_piece`: `Piece` オブジェクト
*   `self.colors`: `dict` (各テトリミノの配色定義)

**Methods (メンバ関数):**
*   `__init__(self, root)`: 初期化、UI構築、イベントバインド。
*   `start_game(self)`: ゲームループの開始（`after` メソッドによる再帰呼び出し）。
*   `handle_keydown(self, event)`: キー入力のディスパッチ。
*   `drop_piece(self)`: ピースを下に落とすロジック。
*   `rotate_piece(self)`: ピースの回転と衝突判定。
*   `check_collision(self, next_pos, next_rotation)`: 衝突判定のコアロジック。
*   `clear_lines(self)`: ライン消去、スコア更新、グリッドのシフト。
*   `reset_game(self)`: 全変数の初期化。
*   `update_ui(self)`: スコア、レベル、キャンバスの再描画。

#### Class: `Piece`
テトリミノの形状と色を管理するクラス。

**Attributes:**
*   `self.shape`: `List[List[int]]` (行列による形状定義)
*   `self.color`: `str` (16進数カラーコード)
*   `self.x`, `self.y`: 現在の座標

### 3.2 データ構造設計

#### 3.2.1 形状定義 (Tetromino Shapes)
各ピースの形状は、2次元リスト（行列）として定義する。
例 (`'I'` ピース):
```python
[[0, 1, 0, 0],
 [0, 1, 0, 0],
 [0, 1, 0, 0],
 [0, 1, 0, 0]]
```

#### 3.2.2 グリッド管理
`self.grid[row][col]` の形式で、各セルに配置されたブロックの「色」を保持する。
*   `None`: 空のセル
*   `"#FF0000"`: 赤色のブロック

---

### 3.3 修正・実装における技術的制約 (Implementation Constraints)

#### 3.3.1 Syntax & Error Prevention (構文エラー防止策)
*   **String Literals**: 全ての文字列（カラーコード含む）は、開始と終了のクォートが一致していることを確認すること。
*   **Variable Naming**: 全角文字（`・`）や、`self.rows = 2ta_rows` のようなタイポを排除し、`self.rows = 20` と明確に定義すること。
*   **Method Naming**: `self.get_new_piece()` 等、標準的なスネークケースを使用し、存在しないメソッドの呼び出しを禁止する。
*   **Argument Uniqueness**: `tk.Label(..., bg="#222", bg="#333")` のように、同一引数を複数回記述しないこと。

#### 3.3.2 Collision & Game Over Logic (ロジック整合性)
*   **Collision Check Flow**:
    1.  移動・回転の「仮」計算。
    2.  新しい座標での `check_collision` 実行。
    3.  衝突した場合、移動をキャンセルし、もし「下方向への移動」の衝突であれば、ブロックを固定（定着）させる。
    4.  ブロック固定後、ライン消去判定 → ピースの新規生成 → 生成時に衝突していれば `Game Over` 確定。

#### 3.3.3 UI Rendering (描画制御)
*   **Canvas Cleanup**: 毎フレームの描画前に `canvas.delete("all")` を実行し、描画の重なり（残像）を防ぐ。
*   **Overlay**: `game_over` フラグが `True` の場合、既存の描画の上に `canvas.create_text` でメッセージを重畳表示する。

## 4. テストケース (Test Cases)

| テスト項目 | 期待される結果 |
| :--- | :--- |
| 起動時テスト | エラーなくウィンドウが立ち上がり、ピースが上部から現れる。 |
| 移動・回転テスト | 左右移動、回転が、壁や既存ブロックに干渉なく動作する。 |
| ライン消去テスト | 横一列が揃った際、スコアが増加し、上のブロックが降りてくる。 |
| Game Over テスト | 画面上部にブロックが積み上がり、次出現時に動けない状態で「Game Over」が表示される。 |
| リセットテスト | `R` キー押下後、スコアが0に戻り、フィールドがクリアされて再開される。 |
